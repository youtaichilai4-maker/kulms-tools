# ADR-0004: KULMS Deadline Notifier の UI をサイドパネル方式に採用

- **Status**: accepted
- **Date**: 2026-04-09
- **Deciders**: yutaakase, Claude

## Context

KULMS（Sakai v23）の課題締切通知拡張機能の UI 方式を決定する必要があった。初期実装ではページ上部に固定バナーを表示する方式だったが、Comfortable PandA（miniPandA）の UI を参考にしたいという要望があった。

Sakai v23 のヘッダー構造は `<ul id="sakai-system-indicators">` にアイコンが `<li>` として並ぶ形式（ヘルプ → サイト一覧 → 検索 → 通知ベル → アカウント名）。

## Decision

ページ上部の固定バナーを廃止し、**Sakai ヘッダー右上にトグルボタンを注入 → 右サイドパネルで課題一覧を表示**する方式を採用する。

### 具体的な実装:
- `#sakai-system-indicators` の通知ベル `<li>` の手前に時計アイコン付き `<li>` を挿入
- 24時間以内の締切件数を赤バッジで表示
- クリックで右からスライドインするサイドパネルを開閉
- パネル内は締切期間でグループ化（24h以内 / 3日以内 / 7日以内 / 14日以内 / 14日以上）
- 各カードに授業名・締切日時・残り時間・課題/クイズのタグを表示
- カードクリックで提出ページに遷移

## Consequences

**メリット:**
- Sakai の既存 UI に自然に統合される（ヘルプ・検索・通知と並ぶ）
- ページコンテンツを押し下げない（バナー方式の問題を解消）
- Comfortable PandA ユーザーに馴染みのある操作感
- グループ化表示で緊急度が一目でわかる

**デメリット:**
- Sakai の DOM 構造に強く依存（`#sakai-system-indicators` セレクタ）
- Sakai のバージョンアップで DOM が変わると壊れる可能性
- フォールバック（固定ボタン）が必要

**リスク緩和:**
- フォールバックとして `position: fixed` のボタンを右上に配置する実装を維持
- セレクタが見つからない場合でも拡張機能自体は動作する

## Alternatives Considered

### ページ上部固定バナー（初期実装）
- `position: fixed; top: 0` でページ全体の上にバナーを表示
- `body` に `padding-top` を追加してコンテンツを押し下げ
- **却下理由**: Sakai の UI と統合感がなく、ページレイアウトに干渉する。ユーザーが Comfortable PandA 方式を希望

### Chrome 拡張ポップアップのみ
- 拡張機能アイコンをクリックしたときのポップアップだけで表示
- **却下理由**: KULMS ページ内でワンクリックで確認できない。ページ内統合が要件

### Sakai の既存オフキャンバス（通知パネル等）に相乗り
- `offcanvas` パネルとして Sakai の Bootstrap 構造を利用
- **却下理由**: Sakai 内部の Bootstrap インスタンスとの競合リスクが高い。独立した DOM が安全

# ADR-0005: KULMS Deadline Notifier の通知方式（メール送信 vs ブラウザ通知）

- **Status**: proposed
- **Date**: 2026-04-09
- **Deciders**: yutaakase, Claude

## Context

課題締切の一覧を毎日メールで送信する機能の要望があった。拡張機能単体でのメール送信の技術的制約と、代替手段を検討した。

## Decision

未決定。以下の選択肢を検討中。

## Alternatives Considered

### 1. Google Apps Script で毎日メール送信
- GAS から Sakai REST API を叩き、`GmailApp.sendEmail()` で送信
- 毎朝定時にトリガー実行。ブラウザ不要
- **課題**: Sakai のセッション認証（Cookie ベース）を GAS から行う必要がある。ECS-ID のログインフローを GAS 内で再現するか、セッショントークンを定期的に渡す仕組みが必要
- **メリット**: ブラウザが閉じていても動作。メールは見逃しにくい

### 2. 拡張機能から外部メール API（SendGrid / Mailgun）
- 拡張機能の Service Worker から API を叩いてメール送信
- **課題**: ブラウザが開いている時のみ。API キーの管理。無料枠の制限
- **メリット**: 拡張機能内で完結

### 3. 拡張機能から Gmail API（OAuth）
- ユーザーの Gmail アカウントで OAuth 認証し、Gmail API 経由で送信
- **課題**: OAuth フローが複雑。`identity` パーミッション追加。ユーザーに Google アカウント連携の許可を求める
- **メリット**: 外部サービス不要

### 4. ブラウザ通知（chrome.notifications）
- 毎朝（またはブラウザ起動時）にデスクトップ通知で「今日締切の課題が N 件」
- **課題**: ブラウザが起動していないと通知されない
- **メリット**: 実装がシンプル。追加の認証不要。すぐ実装可能

### 5. パネル内「今日の締切」サマリー表示
- KULMS を開いたときにパネル上部で当日の締切をハイライト
- **課題**: KULMS を開かないと見えない
- **メリット**: 追加実装が最小限

## 現時点の推奨

**短期**: 選択肢 4（ブラウザ通知）を実装し、すぐに使える通知を提供
**中期**: 選択肢 1（Google Apps Script）を検証し、Sakai 認証の壁をクリアできれば毎日メール配信を実現
